# 업무공유 웹 시스템 (Task + Schedule + Issue)

기획서 기준으로 구현한 **브라우저 전용 데모 앱**입니다. 서버 없이 동작하며 데이터는 **이 PC의 브라우저 localStorage**에 저장됩니다.

## 실행 방법

1. `index.html`을 Chrome/Edge에서 직접 엽니다.
2. 또는 이 폴더에서 정적 서버를 띄웁니다.  
   - Node가 있다면: `npx serve .`  
   - Python이 있다면: `python -m http.server 8080` 후 브라우저에서 `http://localhost:8080`

## 기능 요약

- **대시보드**: 진행률·지연률 KPI, **팀원별 업무 추진 현황**(담당 건수·상태·지연·완료율), **상태·우선순위 차트**(Chart.js CDN), 지연 업무, 미해결 이슈
- **업무**: 리스트/칸반, 필터, 담당자·우선순위·마감일(직접 입력 / 달력 / 캘린더 항목 선택)·의존 업무, **저장 시 마감일이 캘린더에 자동 반영**(`[마감]` 일정), 지연 표시, 상태 변경 시 활동 로그
- **업무 상세**: 댓글, 파일 첨부(Data URL, **최대 약 1MB**), 변경 이력
- **캘린더**: 월 뷰, **날짜 칸 클릭**으로 해당일 일정 등록, 일정 클릭 시 **편집·저장**(관련 업무가 있으면 **업무 마감일·담당자**와 동기화), 일정마다 **담당자 표시**, 업무 연결, 같은 담당자 기준 **기간 겹침 경고**
- **업무 마감일**: 텍스트·날짜 입력·**달력 열기**·캘린더 일정 **목록 선택**
- **이슈**: 업무 연결, 중요도·상태, 미해결 강조
- **회원관리**: 회원 목록(전원), **Admin**만 회원 추가·역할 변경·삭제·비밀번호 변경
- **설정**: 데모 데이터 재시드, 로컬 데이터 초기화(Admin)

## 참고

- 대시보드 차트는 `index.html`에서 **Chart.js CDN**을 불러옵니다. 오프라인이면 차트만 생략되고 나머지는 동작합니다.
- 첨부는 브라우저 저장 한계가 있으므로 **1MB 이하**를 권장합니다. 실서비스에서는 객체 스토리지를 사용하세요.
- 다른 브라우저·다른 PC와 데이터가 공유되지 않습니다.
